# PkpOpenReview

Displays peer review information for published articles with two view modes: by record (version) and by reviewer. Each round (or reviewer) is a static heading above a flat accordion of review cards — clicking a card expands it inline to show the review content. In the "By Round" view the author response, when present, is the last card of the round's accordion.

The component family is intentionally small:

- `PkpOpenReview` — root: states + the two sort tabs
- `PkpOpenReviewByRound` / `PkpOpenReviewByReviewer` — one per ordering; each owns its section headings and all accordion mechanics, including the card headers
- `PkpOpenReviewReviewDetails` / `PkpOpenReviewReviewContent` — default body of an expanded review (reviewer identity, then comments/form responses)
- `PkpOpenReviewAuthorResponseDetails` / `PkpOpenReviewAuthorResponseContent` — default body of an expanded author response (associated authors, then response text)
- `PkpOpenReviewInProgress` — empty state while reviews are underway
- `PkpOpenReviewSummary` — separate summary card (see its own docs); shares the pinia store

## Review States

The component renders one of four states based on the submission's open peer review status:

- **Not Available** — No public review has been confirmed by a reviewer yet. Only a short "data not available" message is rendered.
- **In Progress No Rounds** — At least one public review has been accepted by a reviewer, but no reviewer has submitted their report yet. Shows the in-progress date with no review rounds to display.
- **In Progress** — At least one public review has been accepted by a reviewer but not all reviews are completed yet. Shows the in-progress date and the recommendation breakdown for reviewers who have already submitted their report.
- **Completed** — All open reviews for the submission have been completed. Shows the completion date, the current published version, and a full breakdown of reviewer recommendations.

## HTML Structure — "By Reviewer" View

The card markup matches the "By Round" view except the second header line shows the round/version label instead of the reviewer name.

The expand/collapse mechanics (the "Read Review" / "Hide Review" indicator) stay component-owned — slots replace the descriptive header content, not the accordion behavior.

The `roundHeader` and `reviewerHeader` slots replace the whole header row, including the default heading element. For accessibility, custom content should render its own heading at the provided `headingLevel` and keep metadata (status, affiliation, counts) outside of it.

## Customizing in Plugins

All Base components are globally available. Override PkpOpenReview by registering your own version:

```javascript
// Override PkpOpenReview with custom review header
pkp.registry.registerComponent('PkpOpenReview', {
	template: `
		<BaseOpenReview v-bind="$attrs">
			<template #reviewHeader="{ review }">
				<div class="my-custom-review-header">
					<strong>{{ review.reviewerFullName }}</strong>
					<span>{{ review.reviewerRecommendationDisplayText }}</span>
				</div>
			</template>
		</BaseOpenReview>
	`,
});
```
