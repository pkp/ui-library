# PkpOpenReviewSummary

A sidebar summary component displaying the overall peer review status for a submission. Shows the review status, current version, reviewer recommendation counts, and an expandable explanation of how decisions are summarized.

<Primary />
<Controls />

## Review States

The component renders one of four states based on the submission's open peer review status:

- **Not Available** — No public review has been confirmed by a reviewer yet. Only a short "data not available" message is rendered.
- **In Progress No Reports** — At least one public review has been accepted by a reviewer, but no reviewer has submitted their report yet. Shows the in-progress date and a "no reports completed" placeholder instead of the recommendation breakdown.
- **In Progress** — At least one public review has been accepted by a reviewer but not all reviews are completed yet. Shows the in-progress date and the recommendation breakdown for reviewers who have already submitted their report.
- **Completed** — All open reviews for the submission have been completed. Shows the completion date, the current published version, and a full breakdown of reviewer recommendations.

## Accessibility Notes

- **Heading hierarchy**: `<h2>` for the component title, `<h3>` for all sub-sections (STATUS, CURRENT VERSION, reviewer count, accordion header). This enables screen reader heading navigation (H key).
- **Recommendation rows**: Plain `<div>` elements instead of `<ul>/<li>` — avoids "list, 3 items" + "list item X of Y" verbosity for a short summary. Each recommendation row contains separate spans for label, separator, and count. The separator span (`aria-hidden="true"`) is hidden from screen readers, so assistive technology reads "Approved 1" rather than "Approved – 1".
- **Accordion**: PkpAccordion (built on reka-ui) handles its own ARIA attributes automatically.
